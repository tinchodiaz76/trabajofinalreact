El proyecto ser√° un inventario de productos

Este backend guarda productos que se agreguen en un archivo "memory.json" en la carpeta del backend

Estos productos se pueden agregar y editar

/files

Tipo: GET, devuelve el json con todos los productos

/add

Tipo: POST, header requiere "content-type: application/json", recibe un json con el producto y le inserta un id incremental

/edit

Tipo: POST, header requiere "content-type: application/json", recibe un json con el producto y debe tener una propiedad "id" con el id del producto a editar

/register

Tipo: POST, header requiere "content-type: application/json", recibe un json con user y password para registrarlo en la memoria

/login

Tipo: POST, header requiere "content-type: application/json", recibe un json con user y password y devuelve ok si los datos validan o error si no validan
